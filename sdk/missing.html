Status: 138 images could not be retrieved from the Wayback Machine archive. These images were part of the original ez.no developer documentation for eZ Publish 3.1 (now Exponential CMS). Where images appear in the documentation, they have been replaced with <pre class="missing-image">TODO</pre> blocks describing what the image showed and how to recreate it.

How to Restore These Images

  1. Install Exponential 3.1 (the distribution formerly known as eZ Publish 3.1). A copy is available at exponential.earth.
  2. Run Doxygen on the PHP source to regenerate the 77 class inheritance diagrams. Place the output PNGs in /doc/generated/html/ relative to this site root.
  3. Use a UML tool (e.g. draw.io, PlantUML, or Doxygen) to recreate the architectural UML diagrams in /doc/design/uml/.
  4. Take admin UI screenshots of the Exponential admin interface to replace the tutorial and user manual screenshots. Use the descriptions below as a guide for exactly which screen to capture for each image.
  5. Save images to the exact paths listed in column 1 of the table, relative to the site root /web/vh/ezpublish.se7enx.com/doc/doc.sdk.ezpublish.se7enx.com/.
  6. Remove the placeholder blocks: once an image is in place, find the <!-- IMAGE NOT ARCHIVED: path --> comment and the <pre class="missing-image"> block in the HTML and delete them, leaving only the <!-- <img ...> --> restored to a live <img> tag.
